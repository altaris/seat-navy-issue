ESI callback. You should not manually call this.

Upon receiving a notification from EVE SSO, SNI notifies the appropriate app by issuing a POST request to the predefined app callback, using the PostCallbackEsiOut model.

sni.api.routers.sde.get_sde_category(data: List[int], _tkn: sni.uac.models.Token = Depends(from_authotization_header_nondyn))[source]¶

Given a list of SDE category ids, returns a dict with informations about these types. The return dict key set is guaranteed to be the same as the input list (up to repetition of keys).

Example: Sending a request with [-1, 7, 8, 9] returns:

{
    "8": {
        "category_name": "Charge"
    },
    "9": {
        "category_name": "Blueprint"
    },
    "-1": null,
    "7": {
        "category_name": "Module"
    }
}
